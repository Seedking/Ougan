# 可行性分析
## 工程
### 概念
* 空间 - 标记空间，游戏主程序触发后加载空间内所有音频文件，也可以卸载
* 效果器 - 标记其本身和控制方式
* 轨道 - 标记挂载效果器和输入输出信号，以及轨道播放pan和音量
* 音频 - 标记音频播放的轨道，以及播放时总音量和ADSR（感觉有冲突）
* 事件 - 标记一个事件，可以被游戏主程序触发
* 操控器 - 标记通过特定事件触发的操控器，可以做出对应的操控
* 传递器 - 和游戏交互，获得游戏主程序的某种空间参数（懒得做）
## 需求
### 功能
有图形界面编辑器  
可以将编辑的数据记录为JSON（工程）  
可以加载JSON（工程） 
### 界面
界面模仿StudioOne，轨道信号逻辑模仿Reaper
